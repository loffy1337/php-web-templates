# php-web-templates

Набор моих PHP-заготовок для веб-проектов.

Этот репозиторий нужен как личная коллекция шаблонов, чтобы не собирать с нуля одни и те же базовые части сайта в каждом новом проекте. Здесь я храню простые рабочие модули, которые можно быстро взять за основу, подключить и адаптировать под конкретную задачу.

Сейчас в проекте уже есть две заготовки:

- `router/` - модуль для роутинга и обработки HTTP-запросов
- `db/` - модуль для работы с MySQL через `mysqli`

В дальнейшем репозиторий будет пополняться и другими типовыми заготовками для веб-разработки.

## Для чего нужен проект

- Чтобы не писать рутинную инфраструктуру заново в каждом проекте
- Чтобы ускорять старт новых PHP-сайтов и сервисов
- Чтобы держать под рукой собственные понятные модули без лишней магии
- Чтобы собирать личную библиотеку переиспользуемых решений

## Что есть сейчас

### `router/`

Класс `Router` для регистрации маршрутов и обработки входящих HTTP-запросов.

Что умеет:

- Регистрировать маршруты через `add_route($uri, $handler, $methods)`
- Поддерживать HTTP-методы `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`
- Поддерживать `ANY` для всех доступных методов
- Поддерживать параметры в URI, например `/users/{id}`
- Возвращать `404`, если маршрут не найден
- Возвращать `405`, если маршрут найден, но метод не разрешен
- Возвращать `500` в случае исключения внутри обработчика
- Автоматически отдавать `array` как JSON
- Автоматически отдавать `string` как HTML
- Писать ошибки в `router_errors.log`, если включен debug-режим

Быстрый пример:

```php
<?php
declare(strict_types=1);

require_once __DIR__ . "/router/router.php";

$router = new Router(true);

$router->add_route("/", function () {
    return "<h1>Hello, world!</h1>";
}, ["GET"]);

$router->add_route("/api/users/{id}", function ($params) {
    return [
        "status" => "success",
        "user_id" => $params["id"]
    ];
}, ["GET"]);

$router->route();
```

Примечания:

- В обработчик передается массив параметров маршрута
- Если обработчик возвращает `null`, ответ можно сформировать вручную
- В debug-режиме ошибки дополнительно пишутся в лог

### `db/`

Класс `MySQLDatabase` для удобной работы с MySQL через `mysqli`.

Что умеет:

- Подключаться к базе в конструкторе `MySQLDatabase($database, $username, $password, $host, $port)`
- Автоматически выставлять кодировку `utf8mb4`
- Проверять соединение перед запросом и переподключаться при необходимости
- Выполнять произвольные SQL-запросы через `query($sql, $params = [])`
- Получать набор строк через `select()`
- Получать одну строку через `get_row()`
- Получать одно значение через `get_value()`
- Добавлять данные через `insert()`
- Обновлять данные через `update()`
- Удалять данные через `delete()`
- Очищать таблицу через `truncate()`
- Отдавать последнюю ошибку через `get_error()`

Быстрый пример:

```php
<?php
declare(strict_types=1);

require_once __DIR__ . "/db/db.php";

$db = new MySQLDatabase("my_database", "root", "password");

$insert_id = $db->insert("users", [
    "email" => "user@example.com",
    "name" => "Alex"
]);

if ($insert_id === false) {
    $error = $db->get_error();

    if ($error !== null) {
        echo $error;
    }

    exit;
}

$user = $db->get_row("users", ["id", "email", "name"], "id = ?", [$insert_id]);

if ($user === false) {
    $error = $db->get_error();

    if ($error !== null) {
        echo $error;
    } else {
        echo "User not found";
    }
} else {
    var_dump($user);
}
```

Примечания:

- Значения в запросах нужно передавать через плейсхолдеры `?` и массив `$params`
- Имена таблиц и колонок валидируются отдельно
- Методы чтения возвращают массивы данных, а методы изменения данных возвращают `insert id` или количество затронутых строк
- При ошибке методы возвращают `false`, а для ошибок соединения и выполнения запроса можно дополнительно проверить `get_error()`
- `get_error()` возвращает текущую ошибку и сразу очищает её, поэтому в коде лучше вызывать этот метод один раз и сохранять результат в переменную

## Требования

- PHP 8.0+
- Расширение `mysqli`
- MySQL или MariaDB для модуля `db/`

## Как использовать этот репозиторий

Это не фреймворк и не Composer-пакет, а набор моих локальных рабочих шаблонов.

Обычный сценарий использования такой:

1. Взять нужный модуль из репозитория
2. Подключить его в новый проект
3. При необходимости доработать под конкретную задачу
4. Использовать как основу, а не как жестко зафиксированную библиотеку
