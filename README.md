# php-web-templates

Набор моих PHP-заготовок для веб-проектов.

Этот репозиторий нужен как личная база шаблонов, чтобы не писать с нуля одни и те же рутинные части сайта: обработку HTTP-запросов, маршруты, вспомогательные функции и другие повторяющиеся куски, которые почти в каждом проекте нужны снова.

Сейчас здесь есть только обработчик HTTP-запросов, но дальше репозиторий будет пополняться новыми заготовками для быстрого старта веб-разработки.

## Зачем нужен проект

- Чтобы не собирать базовую инфраструктуру сайта с нуля каждый раз
- Чтобы держать под рукой простые и понятные заготовки без лишней магии
- Чтобы ускорять старт новых PHP-проектов
- Чтобы постепенно собрать собственную библиотеку типовых решений

## Что есть сейчас

На текущий момент в репозитории есть один шаблон:

- `router/` - простой HTTP-роутер для обработки запросов

## Возможности роутера

- Регистрация маршрутов через `add_route($uri, $handler, $methods)`
- Поддержка нескольких HTTP-методов
- Поддержка `ANY` для обработки всех доступных методов
- Параметры в URI, например `/users/{id}`
- Автоматические ответы `404 Not Found` и `405 Method Not Allowed`
- Возврат `array` как JSON-ответа
- Возврат `string` как HTML-ответа
- Логирование ошибок в `router_errors.log`, если включен режим отладки

## Структура

```text
.
|-- README.md
`-- router/
    |-- helper.php
    `-- router.php
```

## Быстрый пример использования

После подключения роутера можно зарегистрировать маршруты и запустить обработку:

```php
<?php
declare(strict_types=1);

add_route("/", function () {
    return "<h1>Hello, world!</h1>";
}, ["GET"]);

add_route("/api/ping", function () {
    return [
        "status" => "success",
        "message" => "pong"
    ];
}, ["GET"]);

add_route("/users/{id}", function ($params) {
    return [
        "status" => "success",
        "user_id" => $params["id"]
    ];
}, ["GET"]);

route();
```

## Как работает обработчик

1. Маршрут добавляется через `add_route()`
2. URI нормализуется и преобразуется в регулярное выражение
3. При вызове `route()` роутер сравнивает текущий запрос со всеми зарегистрированными маршрутами
4. Если маршрут найден, вызывается обработчик
5. Если обработчик возвращает массив, клиент получает JSON
6. Если обработчик возвращает строку, клиент получает HTML
7. Если маршрут не найден, возвращается `404`
8. Если метод не разрешен, возвращается `405`

## Требования

- PHP 8.0+

## Идея репозитория

Это не фреймворк и не универсальный пакет для всех случаев, а набор моих рабочих заготовок, которые можно быстро брать за основу и адаптировать под конкретный сайт или сервис.

## Статус

Проект находится в стадии наполнения. Сейчас основа репозитория - это роутер, а дальше сюда будут добавляться новые шаблоны для типовых задач веб-разработки.
